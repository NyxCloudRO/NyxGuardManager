# NyxGuard Manager 4.0.14

Release date: 2026-07-15

## Highlights

- Dedicated multi-site WireGuard VPN Client workspace in the main sidebar.
- Independent tunnel interfaces, connect/disconnect controls, auto-reconnect, status, handshakes, transfer counters, and per-site ping tests.
- Safe profile validation, explicit full-tunnel route replacement, and overlap protection.
- Guided Add VPN site workflow with clear readiness feedback and active connection controls.
- Preflight detection for remote routes that collide with networks already attached to NyxGuard.
- Dedicated least-privilege VPN agent; the manager container remains unprivileged.
- Existing 4.0.13 installations gain the VPN agent automatically through the general updater.

```bash

```

## General update command

Interactive:

```bash
curl -fsSL https://raw.githubusercontent.com/NyxCloudRO/NyxGuardManager/main/update.sh | sudo bash
```

Explicit non-interactive update to 4.0.14:

```bash
curl -fsSL https://raw.githubusercontent.com/NyxCloudRO/NyxGuardManager/main/update.sh \
  | sudo env FORCE_TAG=4.0.14 NYXGUARD_AUTO_YES=1 bash
```

The update preserves the existing database, certificates, proxy configuration, and other named volumes. It also installs or repairs the VPN agent Compose overlay and reboot-persistent systemd configuration.

The refreshed 4.0.14 installer/updater first verifies `/dev/net/tun`. If a restricted host such as Proxmox LXC does not expose TUN, NyxGuard Manager and its database remain healthy while the VPN agent is skipped with actionable remediation. After the hypervisor passes TUN through, rerun the same update command to enable the agent.

## Verify after update

```bash
docker ps --filter name=nyxguard
docker logs --tail=100 nyxguard-manager
docker logs --tail=100 nyxguard-vpn-agent
```

Expected services on a VPN-capable host:

- `nyxguard-manager`
- `nyxguard-vpn-agent`
- `nyxguard-db`

Without `/dev/net/tun`, the safe expected state is a healthy manager and database with VPN Client unavailable until the host prerequisite is corrected.

See the [VPN Client networking guide](../VPN_CLIENT.md) and the project [changelog](../../CHANGELOG.md) for the full behavior, requirements, security model, and known networking limitation.
